# Health Check Bot

Телеграм-бот, который периодически спрашивает участников «На связи?» и отправляет в общий чат отчёт: кто на связи, кто не отвечает. Если человек молчит совсем долго — бот пингует в группе администратора.

Развёрнут на Vercel (бесплатный тариф Hobby), данные хранятся в Vercel KV (Redis от Upstash).

---

## Что бот делает (простыми словами)

1. Каждый человек один раз регистрируется в личке у бота: вводит имя, выбирает часовой пояс, выбирает «активное окно» (часы суток, в которые бот может писать) и интервал проверки (каждые 4 / 8 / 12 / 24 / 48 / 72 часа).
2. С выбранной периодичностью бот пишет ему «На связи?» и кнопку «Да».
3. Человек жмёт «Да» — в общую группу прилетает «✅ ИМЯ на связи!»
4. Если человек не нажимает кнопку слишком долго (≈ 1/8 от его интервала; для 4 часов это 30 минут, для 24 часов — 3 часа) — в группу падает «❌ ИМЯ **не отвечает**!», и бот отправляет ему повторную проверку.
5. Так бот пробует 3 раза. Если и после третьего повтора молчание — в группу падает большое сообщение с пингом администратора: `@админ ❌❌❌ ИМЯ не отвечает на проверки!`.
6. Если человек заблокировал бота — его автоматически удалит из базы.
7. Все смены состояния «подключился / поставил на паузу / возобновил» тоже сообщаются в группу.

## Часть 1. Подготовка (один раз)

### 1.1. Создать бота в Telegram

1. В Telegram найдите [@BotFather](https://t.me/BotFather).
2. Напишите ему `/newbot`, дайте имя и username для бота.
3. BotFather пришлёт строку вида `123456789:ABC...` — это **`BOT_TOKEN`**. Сохраните.
4. Сразу же отправьте BotFather команду `/setprivacy` → выберите вашего бота → `Disable`. Это нужно, чтобы бот мог читать сообщения в группе (но в нашей задаче он только пишет в группу — `Disable` всё равно безопаснее, чтобы команды работали корректно).

### 1.2. Создать группу и узнать её ID

1. Создайте обычную или супер-группу в Telegram, добавьте туда бота и сделайте его **администратором** (чтобы он мог писать в группе).
2. Самый простой способ узнать ID группы:
   - Добавьте в группу бота [@userinfobot](https://t.me/userinfobot).
   - Он напишет ID группы. Для супергрупп ID начинается с `-100…`, для обычных — с `-…`.
   - После этого `@userinfobot` можно удалить из группы.
3. Запомните это значение — это **`GROUP_CHAT_ID`** (число с минусом).

### 1.3. Решить, кто администратор

Это тот человек, которого бот будет пинговать в группе при полном отсутствии ответа. Узнайте его Telegram-username (например `ivan_petrov`). Сохраните **без `@`** — это **`ADMIN_USERNAME`**.

## Часть 2. Деплой на Vercel

### 2.1. Подготовить репозиторий

Код уже в репозитории: <https://github.com/NOCHNOY-224/health_check_bot>. Если вы клонировали к себе и меняли что-то — запушьте изменения. Ничего больше делать не нужно.

### 2.2. Создать проект на Vercel

1. Зайдите на <https://vercel.com> и войдите через GitHub.
2. Нажмите **Add New… → Project**.
3. В списке репозиториев выберите `health_check_bot`. Нажмите **Import**.
4. На экране настройки проекта оставьте всё по умолчанию (Framework Preset: `Other`).
5. Раскройте **Environment Variables** прямо на этом экране и добавьте 5 базовых переменных (без переменных KV — их добавит сам Vercel чуть позже):

| Имя | Значение |
|---|---|
| `BOT_TOKEN` | токен из BotFather (шаг 1.1) |
| `GROUP_CHAT_ID` | ID группы с минусом (шаг 1.2) |
| `ADMIN_USERNAME` | ник админа БЕЗ `@` (шаг 1.3) |
| `WEBHOOK_SECRET` | любая длинная случайная строка (например 32 символа) |
| `CRON_SECRET` | любая длинная случайная строка (другая) |

> Сгенерировать случайную строку можно на <https://generate-secret.vercel.app/32> или просто стукнуть лбом по клавиатуре пару раз.

6. Нажмите **Deploy**. Подождите 1–2 минуты. После «Deployment Completed» у вас будет URL вида `https://health-check-bot-xxxxx.vercel.app`. **Бот пока работать не будет** — нет базы и webhook не зарегистрирован. Это нормально.

### 2.3. Подключить базу данных (Vercel KV / Upstash Redis)

Это можно сделать только **после** первого деплоя.

1. В левом меню проекта выберите **Storage**.
2. Нажмите **Create Database** → выберите **Upstash for Redis** (это и есть «новый» Vercel KV).
3. Дайте имя (например `health-check-kv`), регион — любой ближний, тариф — `Free`.
4. После создания нажмите **Connect to Project** → выберите ваш проект → подтвердите. Vercel автоматически добавит переменные `KV_REST_API_URL` и `KV_REST_API_TOKEN` (или `UPSTASH_REDIS_REST_URL` / `UPSTASH_REDIS_REST_TOKEN` — обе пары поддерживаются).
5. **Важно:** сделайте **Redeploy** — в разделе Deployments → ⋯ на свежем деплое → **Redeploy** → подтвердить **без** галочки «Use existing Build Cache», иначе новые переменные не подхватятся.

### 2.4. Зарегистрировать webhook и команды бота (один раз)

После того как redeploy с базой прошёл, откройте в браузере:

```
https://<ваш-url>.vercel.app/api/setup?secret=<значение CRON_SECRET>
```

Если вернулось `"ok": true` — webhook зарегистрирован, меню команд в Telegram настроено. Этот шаг нужно повторять только если домен поменялся или вы вручную сбросили webhook.

### 2.5. Настроить периодический запуск (cron-job.org)

> **Почему так:** на бесплатном тарифе Vercel (Hobby) встроенный Cron работает только раз в сутки. Нам нужно тикать каждые 5 минут, поэтому используем бесплатный внешний сервис. Это никак не сказывается на работе — Vercel-функция `/api/cron` остаётся той же.

1. Зайдите на <https://cron-job.org/en/> и зарегистрируйтесь (это бесплатно).
2. В меню **CronJobs → Create cronjob**.
3. Заполните:
   - **Title:** `health-check-bot tick`
   - **URL:** `https://<ваш-url>.vercel.app/api/cron`
   - **Schedule:** Every `5` minutes (или «Every 5 minutes» в шаблоне).
4. Откройте раздел **Advanced → Headers** (или «Custom request headers»). Добавьте заголовок:
   - Name: `Authorization`
   - Value: `Bearer <значение CRON_SECRET>`
5. Сохраните. Cron-job.org начнёт сразу же дёргать ваш `/api/cron` каждые 5 минут.
6. Проверьте: в истории cron-job.org должны идти ответы `200 OK` с JSON `{"ok":true,...}`.

> Альтернатива, если по каким-то причинам не хотите внешний сервис: можно поставить Upstash QStash, или просто перейти на Vercel Pro и использовать встроенный Cron — в этом случае верните в `vercel.json` блок `"crons": [{ "path": "/api/cron", "schedule": "*/5 * * * *" }]`. Никакого другого кода менять не надо.

### 2.6. Готово

Откройте своего бота в Telegram и напишите ему `/start`. Дальше — см. инструкцию для пользователей ниже.

## Часть 3. Инструкция для людей в группе (простым языком)

### Регистрация (один раз)

1. Откройте бота в Telegram (по ссылке, которую вам пришлёт администратор).
2. Нажмите **«Начать»** или отправьте `/start`.
3. Бот спросит имя — напишите своё имя обычным сообщением.
4. Бот покажет кнопки часовых поясов: `МСК`, `МСК+1`, …, `МСК+8`. Нажмите свой.
   - МСК — это Москва.
   - МСК+2 — это, например, Екатеринбург.
   - МСК+5 — это Иркутск. И так далее.
5. Бот покажет кнопки с часами — выберите, **во сколько утром** бот может начать вам писать. Например `09:00`.
6. Затем — кнопки часов **окончания** активного окна. Например `22:00`.
   - Между этими часами бот будет вас тревожить. После 22:00 он замолкнет до утра.
7. Бот покажет кнопки интервалов: `4`, `8`, `12`, `24`, `48`, `72` ч. Выберите, как часто хотите получать «На связи?».

Всё! С этого момента бот будет писать «На связи?» — просто нажимайте «Да».

### Как пользоваться

- **Бот написал «На связи?»** → нажмите «Да». Всё.
- **Я хочу поменять интервал** → `/set_interval`.
- **Я переехал, у меня сменился часовой пояс** → `/set_timezone`.
- **Я хочу поменять часы, когда бот может писать** → `/set_window`.
- **Я в командировке / отпуске, поставьте на паузу** → `/stop`. Группа увидит «🔴 Имя остановил проверки».
- **Я снова в строю** → `/resume`. Группа увидит «🟢 Имя возобновил проверки».
- **Покажи мои настройки** → `/status`.
- **Помогите** → `/help`.

### Что значат сообщения в группе

| Сообщение | Что означает |
|---|---|
| 🟢 Иван подключился к проверкам с интервалом 4 ч. | Иван прошёл регистрацию и подключился. |
| 🟢 Иван возобновил проверки, интервал 4 ч. | Иван был на паузе, теперь снова на связи. |
| 🔴 Иван остановил проверки. | Иван временно отключился. |
| ✅ Проверка 4 ч: Иван на связи! | Иван ответил на проверку. Всё хорошо. |
| ❌ Проверка 4 ч: Иван **не отвечает**! | Иван молчит больше 1/8 интервала. Бот ему отправит повторное сообщение. |
| @админ ❌❌❌ Иван не отвечает на проверки! | Иван не ответил **три повтора подряд**. Дело может быть серьёзным — стоит позвонить или приехать. |

## Часть 4. Если что-то сломалось

### Бот не отвечает в Telegram

1. Зайдите в Vercel → ваш проект → **Deployments** → откройте последний → **Functions**.
2. Кликните `api/webhook` — увидите логи.
3. Чаще всего проблема — не подхватились переменные окружения. Проверьте **Settings → Environment Variables**, после изменения переменных нужно **Redeploy**.
4. Если webhook слетел — снова откройте `https://<url>/api/setup?secret=<CRON_SECRET>` в браузере.

### Бот не присылает проверки сам

1. Проверьте, что задание на <https://cron-job.org> активно и в его истории идут ответы `200 OK`. Если в истории `401` — значит неверный `CRON_SECRET` в заголовке; если `5xx` — смотрите логи Vercel `api/cron`.
2. Проверить руками: откройте `https://<url>/api/cron?secret=<CRON_SECRET>` в браузере — должен вернуться JSON с `"ok": true` и числом обработанных пользователей.
3. Если запросы доходят, но бот всё равно молчит — скорее всего юзер вне «активного окна» по своему времени или интервал ещё не подошёл (проверьте `/status` в личке у бота).

### Хочу удалить пользователя из базы

Самый простой способ — пользователь сам жмёт «Блок» на бота. На следующей попытке отправки бот словит ошибку и удалит запись.

### Хочу поменять токен, переменные

После редактирования переменных в Vercel **обязательно** сделайте Redeploy (в Deployments → ⋯ → Redeploy). Иначе функции продолжат работать со старыми значениями.

## Часть 5. Локальная разработка (для разработчика)

```bash
npm install
npm run build   # tsc --noEmit, проверка типов
```

Полностью локально гонять бот не нужно — он webhook-based. Можно через [ngrok](https://ngrok.com) пробросить локальный сервер и поменять webhook на ngrok-URL, но это редко требуется.

Структура:

```
api/
  webhook.ts    # принимает обновления от Telegram
  cron.ts       # вызывается Vercel Cron каждые 5 минут
  setup.ts      # одноразово: регистрирует webhook и меню команд
src/
  bot.ts        # все хендлеры команд и кнопок (grammY)
  storage.ts    # CRUD в Vercel KV (Upstash Redis)
  scheduling.ts # расчёт «следующего времени проверки» с учётом окна и TZ
  checker.ts    # тело cron — рассылка проверок и эскалаций
  telegram.ts   # обёртки sendMessage с обработкой 403 (бот заблокирован)
  texts.ts      # все текстовые шаблоны
  types.ts      # модель пользователя
```
